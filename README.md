# BranchFlow

BranchFlow is a backend-first ordering and dispatch system designed for multi-branch restaurants.

It replaces WhatsApp-based ordering with a structured transaction engine that gives owners
visibility into orders, deliveries, and branch performance.

## Core Problems Solved
- Lost or delayed orders
- No visibility into branch-level performance
- No delivery tracking or dispatch accountability
- Manual catering inquiries and repetitive phone calls
- No recurring revenue tooling for corporate or group clients

## High-Level Components
- **Backend**: Core transaction engine (orders, dispatch, analytics)
- **Frontend**: Lightweight, mobile-first ordering & dashboards
- **Architecture**: System design, API contracts, and data models

## Status
🚧 Active development — focused on backend reliability and system design.

## Tech Stack (Planned)
- Java / Spring Boot
- PostgreSQL
- REST APIs
- Docker (later)

## Target Users
- Multi-branch restaurants
- Catering businesses
- Corporate & subscription-based food clients
